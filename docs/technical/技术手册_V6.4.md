### A3 数据流全景（17层追踪）

```
物理层: 翅膀运动 → AS5600 → 模拟电压 → ADC引脚
  ↓
采集层 (AS5600_PWM.c):
  L1: AD_Value[0/1] (uint16, 0~4095)
  L2: 限幅滤波 (diff > LIMIT_THRESHOLD? → 丢弃/接受)
  L3: 低通滤波 filtered_ad = α×新值 + (1-α)×旧值
  L4: 类型转换 (uint16_t)filtered_ad
  L5: PROCESS_VALUE宏 (raw - midpoint) & 0x0FFF
  L6: 写入 Wings_Data.Wings_motor[i].Corrective_Angle
  ↓
控制层 (main.c + flight_control.c + motor.c):
  L7: Target_Angle 计算 (midpoint + offset + yaw)
  L8: error = Target - Corrective
  L9: 死区判断 (施密特迟滞)
  L10: 速度规划 (梯形加减速)
  L11: PID计算 Pout + Iout + Dout
  L12: 前馈补偿 ff = FEEDFORWARD_GAIN × Δtarget
  L13: 叠加 final = PID.out + feedforward
  L14: 限幅输出 |final| < motore_max_out?
  ↓
执行层 (motor.c → 硬件):
  L15: Set_Pwm(pwm1, pwm3) → TIM2/TIM3 CCR寄存器
  L16: PWM信号 → H桥驱动器
  L17: 机械运动 → 回到L1 (闭环)
```

## 模块B：核心算法

### B1 双重滤波算法（AS5600_PWM.c）

**解决什么？** AS5600编码器输出有±50~150的随机噪声。

**怎么做？** 两级串联：

```c
// 第一层：限幅门控（去突变）
if (abs(new_value - last_value) > LIMIT_THRESHOLD(80)) {
    limited_value = last_value;  // 异常：丢弃
} else {
    limited_value = new_value;      // 正常：接受
}

// 第二层：指数移动平均EMA（去高频）
filtered_ad[channel] = ALPHA_FLOAT(0.1f) * limited_value 
                       + (1 - ALPHA_FLOAT) * filtered_ad[channel];
```

**效果**: 噪声从±100降到±5（↓95%）

**参数调优**:

| 场景 | LIMIT_THRESHOLD | ALPHA_FLOAT |
|:-----|:---------------:|:------------:|
| 强噪声环境（>150峰值） | 60 | 0.05 |
| 当前默认（平衡） | **80** | **0.10** |
| 高速扑动（>8Hz） | 120 | 0.20 |

### B2 PID + 前馈控制（motor.c + pid.c）

**解决什么？** PID纯反馈导致扑动跟踪延迟15-20°（相位滞后）。

**架构**:

```
Target ──┬──▶ [前馈计算 FF]──┐
           │               │
           ▼               ▼
       ┌───[叠加]───┐
       │           │
Actual─┤[PID闭环] ├────┘
       │           │
       └─────┬─────┘
             ▼
          PWM输出
```

**前馈公式**:
```c
int16_t Calculate_Feedforward(int16_t current_target, uint8_t motor_index)
{
    static int16_t last_target[2] = {0};  // [0]=M1, [1]=M3
    uint8_t idx = (motor_index == 2) ? 1 : 0;
    
    if (!initialized[idx]) {
        last_target[idx] = current_target;
        initialized[idx] = 1;
        return 0;  // 首次调用无输出
    }
    
    int16_t delta = current_target - last_target[idx];
    last_target[idx] = current_target;
    return (int16_t)(FEEDFORWARD_GAIN(3.5) * delta);
}
```

**效果对比**:

| 指标 | 纯PID(V6.0) | PID+前馈(V6.4) | 改善 |
|:-----|:-----------:|:-------------:|:----:|
| 相位滞后 | 18° | 6° | ↓67% |
| RMS误差 | ±30 | ±10 | ↓67% |
| 6Hz跟踪 | 吃力 | 轻松 | ✅ |

### B3 扑动波形生成（flight_control.c Execute_Flap_Step）

**仿生学原理**：
- 下拍快速（35%时间）：产生升力
- 上挥缓慢（65%时间）：节省能量

**Q15波形表（25点）**:
```c
static const int16_t BIOMIMETIC_WAVE[25] = {
    // 下拍阶段（索引0-12，占35%）

    0, -6393, -12540, -18204, -23170, -27146,
    // 极限缓冲（索引13）
    -28900,
    // 上挥阶段（索引14-24，占65%）
    -26384, -22986, -18794, -13856, -8200, -3000,
    2500, 8000, 13500, 20000, 28000
};
```

**状态机驱动**:
```c
sm_idx += sm_dir;         // 步进
if (sm_idx >= 24) sm_dir = -1;  // 到上极限→反向

```

### B5 ⭐ V6.4 新增功能

#### B5.1 自适应PID控制（Adapt_PID_For_Frequency）

**解决什么？** 固定PID无法适应不同频率（低频太慢，高频超调）。

**三频段策略**:

| 频率范围 | Kp | Ki | Kd | 特点 |
|:-------:|:--:|:--:|:--:|:-----|
| ≤3Hz | 18.0 | 0.08 | 3.0 | 激进，响应快 |
| **3-5Hz** | **14.0** | **0.05** | **4.5** | **目标频率，平衡** |
| >5Hz | 12.0 | 0.03 | 5.0 | 保守，稳定 |

**调用时机**: main.c Mode2中每次主循环自动调用。

#### B5.2 动态缓冲区（Get_Dynamic_Buffer_Size）

**解决什么？** 固定缓冲区无法兼顾低频（浪费行程）和高频（保护不足）。

**频率-缓冲映射**:

| 频率 | 动态缓冲 | 对比固定100 |
|:-----:|:-------:|:----------:|
| 2-3Hz | 80 | -20（节省行程）|
| 4Hz | 120 | +20（平衡）|
| 5-6Hz | 160 | +60（强保护）|

**原理**: F = mΔv/Δt，低频惯性小需小缓冲，高频惯性大需大缓冲。

#### B5.3 转向三重滤波（Calculate_Dynamic_Turn）

**解决什么？** Yaw突变导致目标角度跳变→机械冲击。

**三重滤波链路**:
```
原始yaw → [第一重:输入低通α=0.125] → 平滑yaw
         → [第二重:自适应α∈[0.1,0.4]] → 智能平衡
         → [第三重:参数级低通] → 最终turn_ratio
```

**设计思想**: 小转向强滤波（精细），大转向弱滤波（快速响应）。

### D2 常见问题诊断树

```
问题：电机完全不转？
  ├── Switch==0? → 解锁遥控器
  ├── AD_Value==0或4095? → 编码器未连接
  ├── debug_pwm==0? → PID未输出（检查error是否在死区内）
  └── 以上正常? → 检查接线(IN1/IN2)和供电(7.4V)

问题：电机抖动？
  ├── 抖动频率高(>10Hz)? → Kp太大 → 降低Kp
  ├── 在临界值附近? → 迟滞带宽不够 → 增大DEADZONE_EXIT
  └── filtered_ad波动>50? → 滤波不够 → 减小ALPHA_FLOAT

问题：扑动时有"咔咔"声？
  ├── raw_target到极限? → BUFFER太小 → 增大到150
  ├── PWM突然跳变? → Kp太大或转向无滤波 → 降低Kp
  └── 机械间隙? → 手动晃动检查 → 无法软件解决

问题：高频时跟踪跟不上？
  ├── 检查自适应PID是否生效 → Watch观察Kp值
  ├── Kp还是18(固定)? → Adapt_PID未调用 → 检查调用链
  └── Kp已降低但仍跟不上? → 频率太高 → 降低Throttle
```
