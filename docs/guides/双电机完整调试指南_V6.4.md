## Phase 0: 准备工作

### 0.1 硬件连接检查

**编码器ADC输入（双电机版）**:

| 物理位置 | 电机 | STM32引脚 | ADC | Watch变量 |
|:--------:|:----:|:--------:|:---:|:---------|
| 左前 | M3 | PA0 | CH0 | `AD_Value[0]` |
| 右前 | M1 | PA1 | CH1 | `AD_Value[1]` |

**电机PWM输出**:

| 电机 | PWM引脚 | 定时器 | Watch变量 |
|:----:|:-------:|:------:|:---------|
| M1(右前) | PA15+PB3 | TIM2 | `debug_pwm[0]` |
| M3(左前) | PB4+PB5 | TIM3 | `debug_pwm[1]` |

**串口通讯**:

| 功能 | 引脚 | 协议 |
|:-----|:----:|:----|
| USART2_TX | PA2 | CRSF |
| USART2_RX | PA3 | CRSF |

### 0.2 上电检查清单

```
□ 目视检查所有接线无松动/短路
□ 编码器OUT引脚 → PA0(PA1)
□ 电机驱动IN1/IN2 → 对应PWM引脚
□ 电机驱动供电 → 7.4V电池
□ 编码器供电 → 3.3V或5V
□ ELRS接收机 → USART2接口
```

**预期现象**: 电源指示灯亮，电机静止(Switch=0时)

## Phase 1: 极限位置测试 ⭐（必须最先做！）

> ⚠️ **警告：跳过此步骤可能导致机械损坏！**

### 1.1 测试目的

在设置扑动参数之前，必须先确认机械结构的物理极限，避免：
- ❌ 齿轮箱冲击损坏
- ❌ 翅膀碰撞机身
- ❌ 电机过载烧毁

### 1.2 测试步骤

#### 测试上极限（翅膀最高点）

```
1. 遥控器设置：Switch=ON, Mode=0(并翅), Throttle=0, Yaw=0
2. 缓慢增大 midpoint 旋钮（遥控器微调）
3. 观察翅膀上升，记录"即将碰撞"时的值
4. 重复3次取平均
```

**记录表格**:

| 测试项 | 第1次 | 第2次 | 第3次 | 平均值 |
|:-------|:-----:|:-----:|:-----:|:-----:|
| M1上极限 | | | | |
| M3上极限 | | | | |

#### 测试下极限（翅膀最低点）

```
1. 缓慢减小 midpoint 旋钮
2. 观察翅膀下降，记录"即将碰撞"时的值
3. 重复3次取平均
```

| 测试项 | 第1次 | 第2次 | 第3次 | 平均值 |
|:-------|:-----:|:-----:|:-----:|:-----:|
| M1下极限 | | | | |
| M3下极限 | | | | |

#### 测试中点（水平位置）

```
1. 调整 midpoint 使翅膀水平展开
2. 记录此时的编码器值
3. 重复3次取平均
```

| 测试项 | 第1次 | 第2次 | 第3次 | 平均值 |
|:-------|:-----:|:-----:|:-----:|:-----:|
| M1中点 | | | | |
| M3中点 | | | | |

### 1.3 安全计算

```cpp
// 示例计算（请替换为你的实测值）：
// 假设：M1上极限=3200, 下极限=800, 中点=2000

物理行程 = 3200 - 800 = 2400 单位 ≈ 211度
目标振幅(100度) = 1138 单位
单侧振幅 = 569 单位
安全裕度 = 120 单位 (中频缓冲区)

上安全限 = 中点 - 单侧振幅 + 裕度 = 2000 - 569 + 120 = 1551
下安全限 = 中点 + 单侧振幅 - 裕度 = 2000 + 569 - 120 = 2449
实际安全振幅 = 2449 - 1551 = 898 单位 ≈ 79度
```

**判断标准**:
- 实际安全振幅 ≥ **900单位(79度)** → ✅ 可继续
- 否则 → 减小MAX_AMP或调整机械结构

### 1.4 理论极限参考值（代码计算）

```
M1: 上极限≈1098,  下极限≈3357
M3: 上极限≈1018,  下极限≈3277

如果 物理极限 > 理论极限 → 安全 ✅
如果 物理极限 < 理论极限 → 危险！必须减小参数
```

## Phase 2: 编码器校准

### 2.1 什么是中点值？

```
中点值(MIDPOINT) = 翅膀完全水平展开时的编码器读数
每个电机的机械安装都有微小差异，需要单独校准
```

### 2.2 校准步骤

```
1. 手动把2个翅膀掰到水平展开状态
2. 在Keil Watch窗口观察 AD_Value[0](M3) 和 AD_Value[1](M1)
3. 将实测值填入 AS5600_PWM.h 的宏定义：
   #define MOTOR1_MIDPOINT XXXX  // M1(右前) ← AD_Value[1]
   #define MOTOR3_MIDPOINT XXXX  // M3(左前) ← AD_Value[0]
4. 重新编译烧录
```

### 2.3 校准验证

切换到 **Mode1（平翅模式）**，观察：
- ✅ 2个翅膀基本水平（目测误差<5°）
- ✅ `debug_err[0]` 和 `debug_err[1]` 都接近0（±20以内）

**典型范围**: 1500 ~ 2800（当前默认值: **2048**）

## Phase 3: 模式测试

### 3.1 Mode0 并翅模式

**目的**: 验证2个电机能否同步归位到竖直位置

**操作**: Mode=0, Switch=ON

**预期**:
- ✅ 翅膀缓慢收拢到竖直位置（3~5秒）
- ✅ 无明显冲击声
- ✅ 到达后保持静止（允许±5抖动）

**可调参数** ([main.c](Core/Src/main.c)):

| 参数 | 当前值 | 作用 |
|:-----|:------:|:-----|
| SMOOTH_INTERVAL | 5ms | 更新周期 |
| MAX_VELOCITY | 60 | 最大速度 |
| ACCEL_LIMIT | 8 | 加速度限制 |
| HARD_DEADZONE | 20/30 | 硬死区(进/出) |
| SOFT_DEADZONE | 60 | 软死区 |

**异常处理**:
- 不动 → Kp太小 或 中点值错误
- 抖动 → Kp太大 或 死区太小
- 冲击声大 → ACCEL_LIMIT太大

### 3.2 Mode1 平翅模式

**目的**: 验证静态定位精度和Yaw转向

**操作**: Mode=1, Switch=ON

**预期**:
- ✅ 2个翅膀基本水平
- ✅ Yaw左转 → 左翼上抬，右翼下压（丝滑无冲击）
- ✅ Yaw响应及时（延迟<100ms）

**PID参数** (Mode1):
```c
Kp = 10.0, Ki = 0.05, Kd = 2.0
```

**Yaw系数** ([main.c](Core/Src/main.c)):
```c
const int16_t front_targetL = ... + 2*elrs_data.Yaw;  // M3系数=2
const int16_t front_targetR = ... - 2*elrs_data.Yaw;  // M1系数=-2
```

### 3.3 Mode2 扑动模式 ⚠️高风险

**目的**: 验证完整扑动飞行能力

**⚠️ 安全准备**:
```
□ 蝴蝶固定牢固（绳绑/夹具）
□ 周围无人/贵重物品（1米+安全距离）
□ 准备好随时断电
□ 佩戴护目镜
```

**分阶段测试**:

| 阶段 | Throttle | 频率 | 观察 | 时间 |
|:----:|:-------:|:----:|:-----|:----:|
| 1-最低速 | 10 | 3.3Hz | 缓慢扑动，无异常 | 30s |
| 2-低速 | 12 | 4Hz | 明显扑动，有升力感 | 60s |
| 3-中速 | 15 | **5Hz**⭐ | 高频稳定扑动 | 30s |
| 4-高速 | 18 | 6Hz | 极限性能 | 10s⚠️ |

**Mode2 自适应PID（自动切换）**:

| 频率 | Kp | Ki | Kd | 自动切换条件 |
|:-----:|:--:|:--:|:--:|:-----------|
| ≤3Hz | 18.0 | 0.08 | 3.0 | 低频激进 |
| **3-5Hz** | **14.0** | **0.05** | **4.5** | **目标频率** |
| >5Hz | 12.0 | 0.03 | 5.0 | 高频保守 |

**V6.4 扑动关键参数** ([flight_control.c](user/module/flight_control/flight_control.c)):

| 参数 | 当前值(V6.4) | 旧值 | 作用 |
|:-----|:-----------:|:----:|:-----|
| BASE_PERIOD_MS | **3000** | 5000 | 频率基准（改小→更高频）|
| FLAP_CENTER_OFFSET | **250** | 200 | 向下偏移增升力 |
| DOWNSTROKE_RATIO | **35%** | 40% | 下拍更快=更大升力 |
| UPSTROKE_RATIO | **65%** | 60% | 上挥更慢=更省电 |
| MAX_AMP | **1200** | 1100 | 支持~105°振幅 |
| MIN_AMP | **700** | 600 | 防止低频卡死 |
| BUFFER_ZONE_SIZE | **120** | 100 | 极限安全距离 |
| MAX_BIOMIMETIC_FREQ | **7** | 6 | 允许更高频 |

## Phase 4: 参数调优

### 4.1 Keil Watch 完整变量表

**基础组（随时监控）**:

| 变量 | 含义 | 正常范围 |
|:-----|:-----|:-------|
| `elrs_data.Switch` | 安全开关 | 0/1 |
| `elrs_data.Mode` | 当前模式 | 0/1/2 |
| `elrs_data.Throttle` | 油门 | 10-18 |
| `elrs_data.Yaw` | 转向 | -400~+400 |
| `AD_Value[0]` | M3原始ADC | 0~4095 |
| `AD_Value[1]` | M1原始ADC | 0~4095 |

**调试组**:

| 变量 | 含义 | 正常范围 |
|:-----|:-----|:-------|
| `debug_current[0]` | M1校正角度 | -2048~+2048 |
| `debug_current[1]` | M3校正角度 | -2048~+2048 |
| `debug_err[0]` | M1 PID误差 | ±50以内 |
| `debug_err[1]` | M3 PID误差 | ±50以内 |
| `debug_pwm[0]` | M1 PWM输出 | ±6600 |
| `debug_pwm[1]` | M3 PWM输出 | ±6600 |

**高级组（深入分析）**:

| 变量 | 含义 | 观察要点 |
|:-----|:-----|:---------|
| `sm_idx` | 波形表索引 | 0~24循环，24=上极限 |
| `sm_dir` | 扫描方向 | +1=下拍, -1=上挥 |
| `motor_1_pid.Kp` | M1当前Kp | 应随频率自动变化 |
| `Wings_Data.Wings_motor[0].Target_Angle` | M1目标 | 与Corrective的差=跟踪误差 |

### 4.2 症状→方案速查表

| 症状 | 可能原因 | 调哪个文件 | 怎么调 |
|:-----|:---------|:----------:|:-----|
| 电机不转 | 驱动器无供电/信号错 | - | 检查7.4V和接线 |
| 电机单向转 | IN1/IN2接反 | - | 对调信号线 |
| 编码器无读数 | 未供电/OUT断开 | - | 检查3.3V/5V |
| **抖动严重** | **Kp太大/Kd太小** | **motor.c** | **Kp↓ Kd↑** |
| **响应迟钝** | **Kp太小/Ki弱** | **motor.c** | **Kp↑ Ki↑** |
| **冲击噪音大** | **BUFFER_ZONE小** | **fc.c** | **BUFFER↑ 到150** |
| **跟踪跟不上** | **前馈不足/Kp低** | **motor.c** | **FEEDFORWARD_GAIN↑** |
| **高频抖动** | **自适应PID未生效** | **fc.c** | **检查Adapt_PID调用** |
| **转向冲击** | **转向未滤波** | **fc.c** | **确认Calculate_Dynamic_Turn调用** |
| **不同步** | **中点差异大** | **AS5600_PWM.h** | **重新校准MIDPOINT** |
| **升力不足** | **OFFSET/振幅不够** | **fc.c** | **FLAP_CENTER_OFFSET↑ / MAX_AMP↑** |
