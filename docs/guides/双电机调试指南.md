## 二、硬件连接（双电机版）

### 📌 引脚映射表（⚠️ 当前代码实际使用的）

#### 串口通讯（USART2）
| 功能 | STM32引脚 | 外设 | 说明 |
|:----:|:--------:|:----:|:-----|
| **USART2_TX** | PA2 | 接收机TX | 已从PB6迁移 |
| **USART2_RX** | PA3 | 接收机RX | 已从PB7迁移 |

#### 电机驱动（2个）
| 电机 | 物理位置 | PWM引脚 | 定时器通道 | 驱动器类型 |
|:----:|:-------:|:------:|:----------:|:----------:|
| **M1** | 右前 | **PA15** + **PB3** | TIM2 CH1/CH2 | H桥驱动 |
| **M3** | 左前 | **PB4** + **PB5** | TIM3 CH1/CH2 | H桥驱动 |

> ~~M2(右后): PA2+PA3~~ ❌ 已删除（引脚改用于USART2）  
> ~~M4(左后): PB0+PB1~~ ❌ 已删除

#### 编码器ADC输入（2个）
| 物理位置 | 电机编号 | STM32引脚 | ADC通道 | Watch变量 |
|:--------:|:--------:|:--------:|:------:|:---------:|
| **左前** | **M3** | **PA0** | CH0 | `AD_Value[0]` |
| **右前** | **M1** | **PA1** | CH1 | `AD_Value[1]` |

> ~~右后(M2): PA7, CH7~~ ❌ 已删除  
> ~~左后(M4): PA6, CH6~~ ❌ 已删除

### 🔗 完整信号链路（17层追踪）

以 **M1(右前)** 为例：

```
物理翅膀运动 → AS5600磁编码器(PA1) → ADC采样(CH1) 
    ↓
原始值 AD_Value[1] (0~4095)
    ↓
限幅滤波 Limit_Amplitude_Filter()
    ↓
双重滤波 filtered_ad[1] (float)
    ↓
PROCESS_VALUE 宏处理
    ↓
校正角度 Wings_Data.Wings_motor[0].Corrective_Angle (int16)
    ↓
PID计算 motor_1_pid.out
    ↓
前馈补偿 Calculate_Feedforward()
    ↓
PWM输出 debug_pwm[0]
    ↓
H桥驱动器(PA15/PB3)
    ↓
M1电机转动 → 翅膀扑动
```

3. 如果数值变小 → 旋转该编码器磁铁180°

## 四、编码器校准流程

### 4.1 什么是中点值？

```
中点值(MIDPOINT) = 翅膀完全水平展开时的编码器读数

每个电机的机械安装都有微小差异，
所以需要单独测量每个电机的中点值。
```

### 4.2 校准步骤

```
步骤1：手动把2个翅膀都掰到水平展开状态（像蝴蝶停歇时的样子）

步骤2：读取当前的AD_Value原始值
  方法A：看Watch窗口的 AD_Value[0](M3) 和 AD_Value[1](M1)
  方法B：在代码中临时打印

步骤3：填入AS5600_PWM.h中的宏定义
  #define MOTOR1_MIDPOINT XXXX  // M1(右前)的实测值 ← AD_Value[1]
  #define MOTOR3_MIDPOINT XXXX  // M3(左前)的实测值 ← AD_Value[0]

步骤4：重新编译烧录
```

### 4.3 中点值参考范围

```
典型范围：1500 ~ 2800（取决于机械零位）

合理判断：
  • 2个值应该在相近范围内（差距<500算正常）
  • 如果某个值特别大或特别小 → 检查该编码器安装
  • 如果某个值为0或4095 → 编码器故障或未连接
```

### 4.4 校准验证

```
切换到Mode1（平翅模式），观察：
  ✓ 2个翅膀基本保持水平
  ✓ debug_err[0] 和 debug_err[1] 都接近0（误差<±20）
  
如果有偏差：
  • 整体偏高/偏低 → 调整 midpoint 参数（遥控器通道1）
  • 单个翅膀偏差大 → 微调对应MOTORx_MIDPOINT值
```

## 五、模式测试（Mode0/1/2）

### Mode0 并翅模式

**目的**: 验证2个电机能否同时归位到竖直位置

**操作**: Mode=0, Switch=ON

**预期结果**:
- ✅ 2个翅膀缓慢且同步地收拢到竖直位置
- ✅ 归位过程平滑，无明显冲击声
- ✅ 到达目标后保持静止（允许微小抖动±5）
- ✅ 从开始到完成约需 3~5 秒

**可调参数** ([main.c](Core/Src/main.c)):

| 参数 | 当前值 | 作用 | 调整建议 |
|:----:|:-----:|:----:|:--------|
| SMOOTH_INTERVAL | 5ms | 更新周期 | 减小=更快响应 |
| MAX_VELOCITY | 60 | 最大速度 | 增大=更快归位 |
| ACCEL_LIMIT | 8 | 加速度限制 | 减小=更平滑 |
| HARD_DEADZONE | 20/30 | 硬死区(进/出) | 增大=提前停止 |
| SOFT_DEADZONE | 60 | 软死区 | 增大=提前减速 |

**异常处理**:
```
问题：电机不动
  → Kp太小 或 中点值错误
  → 解决：增大Kp到12~15，或重新校准中点

问题：电机抖动
  → Kp太大 或 死区太小
  → 解决：降低Kp到8，或增大DEADZONE

问题：冲击声音大
  → 加速度太大
  → 解决：减小ACCEL_LIMIT到5
```

### Mode1 平翅模式

**目的**: 验证静态定位精度和Yaw转向响应能力

**操作**: Mode=1, Switch=ON

**预期结果**:
- ✅ 2个翅膀基本水平（目测误差<5°）
- ✅ Yaw左转 → 左翼上抬，右翼下压
- ✅ Yaw右转 → 右翼上抬，左翼下压
- ✅ 转向响应及时（延迟<100ms）
- ✅ 无持续抖动或振荡

**PID参数** (Mode1默认):

## 七、常见问题排查

### 7.1 硬件类

| 问题 | 可能原因 | 解决方法 |
|:----:|:-------:|:--------|
| 电机不转 | 驱动器无供电/信号错 | 检查7.4V电源和PWM接线 |
| 电机单向转 | H桥单边损坏/接线错 | 更换驱动器或检查IN1/IN2 |
| 编码器无读数 | 未供电/OUT断开 | 检查3.3V/5V和信号线 |
| 编码器值不变 | 磁铁丢失/距离过远 | 重新安装磁铁（距离<3mm）|
| 电机发烫 | 堵转/Kp过大 | 降低Kp或检查机械卡死 |

### 7.2 软件类

| 问题 | 可能原因 | 解决方法 |
|:----:|:-------:|:--------|
| 翅膀方向全反 | PID输出取反问题 | 检查motor.c右侧取反逻辑 |
| 单个翅膀反向 | 该电机IN1/IN2接反 | 对调驱动器两根信号线 |
| 抖动严重 | Kp过大/Kd过小 | Kp↓ Kd↑ |
| 响应迟钝 | Kp过小/积分弱 | Kp↑ Ki↑ |
| 冲击噪音大 | BUFFER_ZONE小/加速度大 | 增大BUFFER_ZONE_SIZE |
| 不同步 | 中点值差异大 | 重新校准MIDPOINT |

### 7.3 系统类

| 问题 | 可能原因 | 解决方法 |
|:----:|:-------:|:--------|
| 遥控无响应 | ELRS未绑定/通道映射错 | 重新绑定接收机 |
| 串口无输出 | DEBUG_MODE未定义 | 取消注释#define DEBUG_MODE |
| 程序卡死 | motor_test()未删除 | 注释掉motor_test调用 |
| 电池掉压快 | 放电倍率不足/老化 | 更换高倍率电池 |

## 附录B：电机方向测试

使用 `motor_test()` 函数（在main.c第98行）：

```c
void motor_test(void)
{
    Set_Pwm(3000, 3000);  // 给一个正向脉冲（双电机版本：只传2个参数）
    HAL_Delay(100);
    Set_Pwm(0, 0);        // 停止
    while (1);             // 死循环等待
}
```

**使用方法**:
1. 在main()函数末尾调用 `motor_test();`
2. 重新编译烧录
3. **手动把2个翅膀都掰到水平展开状态**
4. 上电运行
5. 观察2个翅膀的运动方向

**预期结果**: 每个电机都应该向下摆动（朝向地面方向）

**如果反向**: 对调该驱动器的IN1和IN2接线

**⚠️ 测试完成后务必注释掉 motor_test(); 调用！**
