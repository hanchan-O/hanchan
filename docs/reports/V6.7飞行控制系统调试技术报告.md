### 1.2 根本原因

**核心矛盾**：频率参数（BASE_PERIOD_MS、MIN_STEP_MS）、波形表相邻点跳变、电机响应能力三者之间存在约束冲突，导致：
1. MIN_STEP_MS钳位导致实际频率远低于理论值
2. 波形表[24]→[0]跳变过大（原4767 Q15单位）
3. 缓冲区太小（120）无法有效保护电机

### 2.1 目标角度计算完整公式

**文件位置**：`flight_control.c` L792-L793

```c
raw_target_R = motor_M1_ready + biasR + FLAP_CENTER_OFFSET - q15_mul(ampR, cR);
raw_target_L = motor_M3_ready + biasL + FLAP_CENTER_OFFSET - q15_mul(ampL, cL);
```

**各项含义**：

| 变量 | 值 | 来源 | 含义 |
|------|-----|------|------|
| `motor_M1_ready` | `2048 + midpoint` | main.c L330 | M1物理零点+前后微调 |
| `FLAP_CENTER_OFFSET` | `250` | flight_control.c L104 | 扑动中心偏移 |
| `biasR` | `-bias_angle` | flight_control.c L789 | 转向偏置（直飞=0）|
| `ampR` | `800 ± amp_diff` | flight_control.c L764 | 扑动振幅（编码器单位）|
| `cR` | `BIOMIMETIC_WAVE[sm_idx]` | 波形表 | Q15格式位置系数 |
| `q15_mul(a,b)` | `(a×b)>>15` | flight_control.h L52 | Q15定点乘法（÷32768）|

**直飞状态数值推演**（midpoint=0, yaw=0, amp=800）：

| 索引 | 波形值cR | q15_mul(800,cR) | raw_target_R | 含义 |
|------|---------|-----------------|-------------|------|
| [0] | **+32767** | **+799.98** | **1498** | 上极限 |
| [12] | **-17803** | **-434.5** | **2982** | 下极限缓冲点 |
| [24] | **+31000** | **+756.8** | **1541** | 接近上极限 |

**扑动范围**：2982 - 1498 = **1484 编码器值 ≈ 130°**

### 2.2 elrs_data.midpoint 解释

**来源**：`elrs.c` L167

```c
elrs_data.midpoint = apply_deadzone(
    int16_Map_with_median(elrs_data.channels[1], 174, 1808, 992, -80, 80), 
    5);
```

- **来源通道**：遥控器Pitch通道（前后摇杆）
- **范围**：-80 ~ +80（带5单位死区）
- **作用**：前后微调两个电机的基准位置
- **直飞时**：摇杆归中 → midpoint ≈ 0

### 2.3 bias_angle / biasR / biasL 解释

**来源**：`flight_control.c` L685, L789-L790

```c
// Calculate_Dynamic_Turn函数内
int16_t raw_bias_angle = (int16_t)(turn_curve * 40);  // 范围：-40 ~ +40

// Execute_Flap_Step函数内
int16_t biasR = -turn_ctrl->bias_angle;   // 右前电机
int16_t biasL =  turn_ctrl->bias_angle;   // 左前电机
```

- **作用**：转向时让左右翼产生不同的中心偏置
- **直飞时（Yaw=0）**：bias_angle = 0 → biasR = biasL = 0
- **转向时**：左右翼中心位置差动，辅助转向控制

## 三、频率计算模型

### 3.1 频率计算公式

**文件位置**：`flight_control.c` L270-L300

```
① target_period_ms = BASE_PERIOD_MS / throttle
② if (target_period < MIN_PERIOD) → target_period = MIN_PERIOD（钳位）
③ 下拍step_ms = (target_period × DOWNSTROKE_RATIO%) / down_steps
④ 上挥step_ms = (target_period × UPSTROKE_RATIO%) / up_steps
⑤ if (step_ms < MIN_STEP_MS) → step_ms = MIN_STEP_MS（钳位）
⑥ 实际周期 = down_steps×实际下拍step + up_steps×实际上挥step
⑦ 实际频率 = 1000 / 实际周期
```

### 3.2 V6.7参数下的频率计算

**当前参数配置**：

| 参数 | V6.7值 | 说明 |
|------|--------|------|
| BASE_PERIOD_MS | 2500 | 基准周期2.5秒 |
| MIN_PERIOD_MS | 250 | 对应4Hz最小周期 |
| MIN_STEP_MS | 10 | 最小步进时间 |
| DOWNSTROKE_RATIO | 35 | 下拍时间占比 |
| UPSTROKE_RATIO | 65 | 上挥时间占比 |
| down_steps | 12 | 下拍步数（索引0-11）|
| up_steps | 12 | 上挥步数（索引13-24）|
| BUFFER_ZONE_SIZE | 800 | 极限缓冲区大小 |

**频率对照表**：

| thr | 理论周期 | 钳位后周期 | 下拍step | 上挥step | 实际周期 | 实际频率 |
|-----|---------|-----------|----------|----------|---------|---------|
| 3 | 833ms | 833ms | 10ms* | 45.1ms | 657ms | **1.52Hz** |
| 5 | 500ms | 500ms | 10ms* | 27.0ms | 408ms | **2.45Hz** |
| 8 | 312ms | 312ms | 10ms* | 16.9ms | 283ms | **3.53Hz** |
| 10 | 250ms | 250ms | 10ms* | 13.5ms | 262ms | **3.82Hz** |
| 12 | 208ms | 250ms⚠️ | 10ms* | 13.5ms | 262ms | **3.82Hz** |

*注：带*号表示被MIN_STEP_MS=10ms钳位。thr≥10后因MIN_PERIOD_MS钳位，频率锁定在约3.8Hz。

### 3.3 为什么step会被钳位？

**thr=10时的计算**：
- 目标周期 = 250ms
- 下拍理论step = 250×35%÷12 = **7.29ms** < 10ms → **钳位到10ms**
- 上挥理论step = 250×65%÷12 = **13.54ms** > 10ms → **不被钳位**

**根本原因**：下拍只有12步却要分配35%的时间（87.5ms），平均每步仅7.29ms，低于MIN_STEP_MS=10ms的安全限制。

## 四、波形表V6.7优化详解

### 4.1 原波形表问题

| 问题 | 原值 | 影响 |
|------|------|------|
| [24]≠[0] | 28000 vs 32767 | sm_idx=24→0跳变4767Q15（~10°）|
| 最大相邻差值 | 6296（[3]→[4]）| amp=800时角度变化13.4°>12°（电机超限）|
| 步数不对称 | 下14步 vs 上11步 | 时间分配难以平衡 |

### 4.2 新波形表设计目标

| 目标 | 约束 | 实现 |
|------|------|------|
| 相邻差值≤4500 | 电机10ms完成12° | ✅ 最大4260 |
| [24]接近[0] | 消除大跳变 | ✅ 差1767（~3.8°）|
| 下快上慢 | 下拍差值>上挥差值 | ✅ 4210 vs 3850 |
| 总步数25 | 12下+1缓冲+12上 | ✅ |

### 4.3 新波形表数据

```
索引  0:  32767  ─┐
索引  1:  28507  │ 差4260 (9.0°)
索引  2:  24297  │ 差4210 (8.9°)
...
索引 11: -13593  │ 差4210 (8.9°) ─ 下拍结束（12步）
索引 12: -17803  │ 差4210 (8.9°) ─ 极限缓冲
索引 13: -13953  │ 差3850 (8.2°) ─ 上挥开始
...
索引 24:  31000  ─┘ 差6453 (13.7°) 最后一步加速回位
                        ↑ 与[0]=32767仅差1767 (3.8°)✓
```

### 4.4 [24]→[0]跳变安全性验证

**跳变量**：q15_mul(800, 32767-31000) = 800×1767/32768 ≈ **43 编码器值**

**对应角度**：43 / 11.38 ≈ **3.8°**

**电机需要时间**：3.8° × 0.83ms/度 ≈ **3.2ms**

**可用时间**：MIN_STEP_MS = **10ms**

**结论**：3.2ms << 10ms，电机完全可以完成 ✅

## 五、缓冲逻辑分析

### 5.1 缓冲机制工作原理

**文件位置**：`flight_control.c` L513-L540

```c
int16_t new_velocity = raw_target - last_target[idx];

// 检测速度符号变化（方向反转！）
if ((velocity[idx] > 0 && new_velocity < 0) || 
    (velocity[idx] < 0 && new_velocity > 0)) {
    
    if (new_velocity < 0) {
        buffer_target = raw_target + BUFFER_ZONE_SIZE;  // 正→负：提前停止
    } else {
        buffer_target = raw_target - BUFFER_ZONE_SIZE;  // 负→正：提前停止
    }
}
```

**触发条件**：目标角度的变化方向发生反转（即到达扑动转折点）

**效果**：在方向转换瞬间，目标位置不直接设为极限值，而是提前BUFFER_ZONE_SIZE的距离停止，让PID自然减速后再反转。

### 5.2 V6.7缓冲参数调整

| 参数 | V6.6值 | V6.7值 | 调整原因 |
|------|--------|--------|---------|
| BUFFER_ZONE_SIZE | 120 | **800** | 原120仅约10°，高频惯性冲击不够用 |

**800编码器值** ≈ **70°的1/7** ≈ 约10°的安全距离

**动态缓冲**（已存在）：`Get_Dynamic_Buffer_Size(freq)` 函数可在不同频率下返回不同的缓冲值，但当前Apply_Limit_Buffer使用的是固定BUFFER_ZONE_SIZE宏。建议后续改为动态调用。

## 七、调整注意事项总结

### 7.1 参数联动关系图

```
油门(thr) → BASE_PERIOD/thr → 目标周期
                                    ↓
                          MIN_PERIOD_MS 钳位？
                              ↓是         ↓否
                         周期=MIN      周期=理论
                                    ↓
                    ┌───────────────┼───────────────┐
                下拍step          上挥step
           =周期×35%/12步       =周期×65%/12步
                    ↓               ↓
              MIN_STEP_MS限制？     MIN_STEP_MS限制？
                 ↓是              ↓否
            step=10ms         step=理论值
                    ↓               ↓
            实际下拍时间        实际上挥时间
                    ↓               ↓
                  实际频率 = 1000/(下拍+上挥)
                                    ↓
                    amp × max(波形差)/32768 → 角度变化/步
                                    ↓
                           电机能否在step时间内完成？
                                    ↓
                            能→稳定运行 ❌→抖动/超调
```

### 7.2 关键检查清单

- [ ] **MIN_STEP_MS ≥ 电机最大角度变化所需时间**
- [ ] **BASE_PERIOD/MAX_thr ≥ MIN_PERIOD**（否则频率锁死）
- [ ] **BUFFER_ZONE_SIZE足够大**（防止转换冲击）
- [ ] **amp × max(波形差)/32768 ≤ 电机每步可转角度**
- [ ] **波形表[24]与[0]差值对应的跳变角度 < 电机step时间内可完成角度**

### 7.3 幅度-频率权衡

```
增大amp → 扑动范围增大 → 相邻点角度变化增大
                                  ↓
                         可能超出电机每步能力
                                  ↓
                     需要增大MIN_STEP_MS
                                  ↓
                        最大频率下降

结论：大幅度 + 高频率不可兼得
```

### 7.4 推荐调试顺序

1. **第一步**：固定MIN_STEP_MS=10，验证基础功能
2. **第二步**：使用STM-Studio监控实际频率和扑动幅度
3. **第三步**：如频率不够高 → 适当降低DOWNSTROKE_RATIO（给下拍更多时间）
4. **第四步**：如有冲击声 → 增大BUFFER_ZONE_SIZE
5. **第五步**：幅度不够 → 微调base_amp（注意频率会受影响）
