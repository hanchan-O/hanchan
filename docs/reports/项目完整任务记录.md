## 二、调试步骤详细记录

### 2.1 阶段1: 基础功能验证（V6.0-V6.2）

**目标**: 确认Mode2基本扑动功能可用

**步骤**:
1. 烧录V6.0固件，切换到Mode2
2. 观察电机是否扑动 → ✅ 扑动但超调严重
3. 检查波形表输出 → 发现[24]→[0]跳变过大
4. 检查极限位置缓冲 → 缓冲区太小(100)

**发现的问题**:
- P1: 极限位置超调严重，有"咔咔"撞击声
- P2: 波形表首尾不连续

**解决方案**:
- 增大BUFFER_ZONE_SIZE从100到120
- V6.2统一编码器方向

### 2.2 阶段2: 频率控制调试（V6.3-V6.4）

**目标**: 修复频率控制异常

**步骤**:
1. 使用STM-Studio监控实际扑动频率
2. 发现：油门中间值时频率最高，两端反而低
3. 分析频率计算公式 → 发现MIN_STEP_MS钳位导致频率异常
4. 重新调参：BASE_PERIOD_MS、MIN_PERIOD_MS、MIN_STEP_MS

**发现的问题**:
- P3: 频率控制异常——理论频率与实际频率不符
- P4: 自适应PID参数不合理

**解决方案**:
- V6.4引入自适应PID策略
- V6.4引入动态缓冲区大小

### 2.3 阶段3: 波形表优化（V6.7-V6.9）

**目标**: 解决波形表跳变和不对称问题

**步骤**:
1. 分析原波形表[24]→[0]跳变：4767 Q15单位 ≈ 10°
2. 计算电机能力：10°需8.3ms，但MIN_STEP_MS=10ms，临界
3. V6.7重写波形表，减小相邻差值到4260
4. V6.8改为对称设计（50%/50%时间分配）
5. V6.9最终修正Q15范围到[-32767, 32767]，严格对称

**关键决策**:
- 放弃"下快上慢"仿生特性，改为对称控制
- 原因：对称设计简化调试，提高电机跟踪可靠性
- 仿生特性可在基础功能稳定后通过时间分配恢复

### 2.4 阶段4: 滤波延时修复（V6.10）

**目标**: 解决编码器数据100ms延时问题

**步骤**:
1. 通过STM-Studio对比原始ADC值和滤波值
2. 发现滤波值比原始值滞后约100ms
3. 计算延时：采样周期/α = 2ms/0.2 = 10ms（理论）
4. 实际延时更大是因为多级滤波叠加
5. 将ALPHA_FLOAT从0.3提升到0.7
6. 验证：延时降至约30ms

**关键发现**: 滤波延时是PID超调的根本原因之一。PID在100ms前的位置上做决策，必然导致超调。

### 2.5 阶段5: 转向响应优化（V6.10）

**目标**: 解决120ms转向延时

**步骤**:
1. 操作偏航摇杆，观察电机响应延迟
2. 发现从摇杆操作到电机响应约120ms
3. 分析Calculate_Dynamic_Turn函数
4. 发现输入滤波α=0.125导致约60ms延时
5. 加上参数级滤波α=0.1-0.4，总延时约120ms
6. 将输入滤波α改为0.5，参数级α改为0.4-0.8
7. 验证：转向延时降至约30ms

### 2.6 阶段6: 模式切换修复（V6.5-V6.10）

**目标**: 解决模式切换时电机异常行为

**步骤**:
1. 从Mode2切换到Mode1，观察电机行为
2. 发现电机有时单向旋转
3. 分析原因：Mode2的Adapt_PID_For_Frequency修改了PID参数
4. 切换到Mode1时PID参数还是Mode2的值
5. V6.5添加PID_clear()调用
6. V6.10添加PID参数强制恢复机制

## 三、问题分析与解决方案

### 3.1 问题1: 极限位置超调严重

**现象**: Mode2扑动时，电机到达极限位置产生"咔咔"撞击声

**根因分析**:
1. 波形表[24]→[0]跳变4767 Q15单位（约10°）
2. 缓冲区BUFFER_ZONE_SIZE=120太小
3. PID参数Kp=18过于激进

**解决方案**:
- V6.7: 重写波形表，消除大跳变
- V6.7: 增大BUFFER_ZONE_SIZE到800
- V6.9: 对称波形表，[24]=[0]=32767
- V6.10: 自适应PID，降低Kp到5.0-6.0

**效果**: 超调从严重改善到轻微

### 3.2 问题2: 频率控制异常

**现象**: 油门中间值时频率最高，两端反而低

**根因分析**:
```
理论频率 = 1000 / (BASE_PERIOD / throttle)
实际频率受MIN_STEP_MS钳位影响：
  - 低油门：step_ms > MIN_STEP_MS → 不钳位 → 频率正常
  - 中油门：step_ms ≈ MIN_STEP_MS → 部分钳位 → 频率异常
  - 高油门：step_ms < MIN_STEP_MS → 全部钳位 → 频率锁死
```

**解决方案**:
- 调整BASE_PERIOD_MS从3500到2500
- 调整MIN_PERIOD_MS从400到250
- 限制油门范围6-10
- V6.10: MIN_STEP_MS从16降到8

**效果**: 频率与油门呈线性关系

### 3.3 问题3: 编码器滤波延时

**现象**: PID跟踪误差大，电机超调

**根因分析**:
- ALPHA_FLOAT=0.3 → 滤波延时约100ms
- PID在100ms前的位置上做决策
- 扑动周期250ms（4Hz），100ms延时占40%周期

**解决方案**:
- ALPHA_FLOAT从0.3提升到0.7
- 延时从约100ms降至约30ms

**效果**: PID跟踪误差显著减小

### 3.4 问题4: 前馈控制导致超调

**现象**: 开启前馈后电机超调更严重

**根因分析**:
- 扑动目标是周期性平滑变化（波形表生成）
- 前馈 = 目标变化量 × FF_GAIN
- 每步目标都变（约8°/步），前馈持续叠加
- 叠加到PID输出上 → 总输出超过需要 → 超调

**解决方案**:
- FEEDFORWARD_GAIN从3.5设为0.0（关闭前馈）

**效果**: 超调消除，纯PID跟踪更稳定

### 3.5 问题5: 模式切换PID参数污染

**现象**: 从Mode2切到Mode1/Mode0后控制异常

**根因分析**:
- Mode2的Adapt_PID_For_Frequency动态修改PID参数
- 切换模式时PID参数未恢复
- Mode1/Mode0使用被Mode2修改过的参数

**解决方案**:
- V6.5: 模式切换时调用PID_clear()
- V6.10: 每次进入Mode1/Mode0时强制恢复PID参数

**效果**: 模式切换后控制正常

### 3.6 问题6: 转向响应延时

**现象**: 操作偏航摇杆后约120ms电机才响应

**根因分析**:
- 输入滤波α=0.125 → 延时约60ms
- 参数级滤波α=0.1-0.4 → 延时约60ms
- 总延时约120ms

**解决方案**:
- 输入滤波α从0.125改为0.5
- 参数级滤波α从0.1-0.4改为0.4-0.8

**效果**: 转向延时从120ms降至约30ms

## 四、Mode2功能实现架构

### 4.1 数据流图

```
遥控器CRSF帧
    │
    ↓ ELRS_UARTE_RxCallback() [elrs.c]
    │
elrs_data结构体
    │ Yaw, Throttle, Mode, Switch, midpoint
    │
    ↓ main.c 主循环
    │
    ├── Mode2入口判断
    │   │
    │   ├── reset_flap_state() [首次进入]
    │   ├── PID_clear() [首次进入]
    │   │
    │   ├── Calculate_Dynamic_Turn(Yaw) → turn_ctrl
    │   │   │
    │   │   └── 输入滤波(α=0.5) → 二次曲线映射 → 参数滤波(α=0.4-0.8)
    │   │
    │   ├── thr限制(6-10)
    │   │
    │   ├── 非阻塞定时判断
    │   │   │
    │   │   ├── Calculate_Flap_Step_Time(thr) → step_ms
    │   │   ├── Execute_Flap_Step(Yaw, turn_ctrl, M3_ready, M1_ready)
    │   │   │   │
    │   │   │   ├── 读取BIOMIMETIC_WAVE[sm_idx]
    │   │   │   ├── q15_mul(amp, wave) → 角度偏移
    │   │   │   ├── 目标角度 = midpoint + bias - q15_result
    │   │   │   ├── 写入Wings_motor[0/2].Target_Angle
    │   │   │   └── sm_idx++ (0→24循环)
    │   │   │
    │   │   ├── step_ms重新计算
    │   │   ├── Adapt_PID_For_Frequency(freq) → 动态PID参数
    │   │   ├── Estimate_Attitude() [每20ms]
    │   │   ├── Motor_Sync_Compensate() [每20ms]
    │   │   └── sm_next_tick += step_ms
    │   │
    │   └── Motor_PID_Control() [每次循环]
    │       │
    │       ├── Calculate_Feedforward() → ff=0 (已关闭)
    │       ├── PID_calc(&motor_3_pid, actual, target) + ff → M3输出
    │       ├── -(PID_calc(&motor_1_pid, actual, target) + ff) → M1输出(取反)
    │       ├── LimitMax() → 输出限幅
    │       └── Set_Pwm(m1, m3) → H桥PWM输出
    │
    ↓
    电机转动 → AS5600编码器反馈
    │
    ↓ StarAndGetResult() [AS5600_PWM.c]
    │
    ADC采样(500Hz) → 限幅滤波(阈值200) → 低通滤波(α=0.7)
    → PROCESS_VALUE(raw, midpoint) → Wings_motor[i].Corrective_Angle
    │
    └──→ 回到PID_calc的ref输入（闭环）
```
