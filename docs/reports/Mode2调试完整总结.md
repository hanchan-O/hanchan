#### 📍 变更1: AS5600_PWM.c - 滤波系数修复

**文件**: `user/module/AS5600/AS5600_PWM.c`
**位置**: 第67-75行 (ALPHA_FLOAT定义)

```c
// 修改前

// 修改后
#define ALPHA_FLOAT 0.7f
```

**必要性**: 原值0.3导致约100ms的滤波延时，使PID控制器接收到的位置反馈严重滞后，Mode2扑动跟踪误差巨大。增大到0.7后延时降至约30ms，显著改善跟踪性能。

**功能目标**: 消除编码器数据采集环节的延时瓶颈，使PID控制器能及时获取电机实际位置。

- 对称设计：上下拍时间相同（50%/50%）

- [24]与[0]完全相同（32767），消除跳变

**功能目标**: 生成稳定、对称、电机可跟踪的扑动波形。

#### 📍 变更4: flight_control.c - 自适应PID参数策略

**文件**: `user/module/flight_control/flight_control.c`
**位置**: 第340-380行 (Adapt_PID_For_Frequency函数)

```c
// 修改前（固定PID参数）
// Mode2: Kp=18, Ki=0.08, Kd=3.0（所有频率统一）

// 修改后（自适应PID）
void Adapt_PID_For_Frequency(float current_freq, ...) {
    if (current_freq <= 3.0f) {
        Kp = 6.0f; Ki = 0.0f; Kd = 2.0f;   // 低频：适度参数
    } else if (current_freq <= 4.0f) {
        Kp = 5.5f; Ki = 0.0f; Kd = 1.5f;   // 中频：平衡参数
    } else {
        Kp = 5.0f; Ki = 0.0f; Kd = 1.0f;   // 高频：保守参数
    }
}
```

**必要性**: 不同频率下电机响应特性差异很大。原固定参数Kp=18在低频时导致超调，在高频时又不够。自适应策略根据频率动态调整，确保各频段都有良好跟踪。

**关键发现**: Ki设为0.0（关闭积分项）是重要优化。扑动是周期性运动，积分项会在一个方向累积后难以在反方向消除，导致单向偏移。

**功能目标**: 在不同扑动频率下都能实现稳定跟踪，避免超调和振荡。

#### 📍 变更5: flight_control.c - 幅度补偿策略优化

**文件**: `user/module/flight_control/flight_control.c`
**位置**: 第305-330行 (Apply_Amplitude_Compensation函数)

```c
// 修改前（高频增大补偿）
// ≤3Hz: 1.00x, 3-5Hz: 1.08x, >5Hz: 1.15x

// 修改后（高频减小幅度，平衡升力和稳定性）
if (current_freq <= 2.5f) {
    compensation_factor = 1.00f;  // 低频：1024（精确90°）
} else if (current_freq <= 3.5f) {
    compensation_factor = 0.97f;  // 中频：约87°
} else if (current_freq <= 4.5f) {
    compensation_factor = 0.94f;  // 高频：约85°
} else {
    compensation_factor = 0.91f;  // 甚高频：约82°
}
```

**必要性**: 原策略是高频时增大振幅补偿衰减，但实际测试发现高频大幅扑动导致电机跟踪不上，反而产生更大超调。新策略是高频时适度减小幅度，让电机能稳定跟踪，虽然单次扑动升力略减，但整体效率更高。

**功能目标**: 平衡扑动幅度与电机跟踪能力，确保高频时也能稳定运行。


#### 📍 变更11: motor.c - 前馈增益关闭

**文件**: `user/module/motor/motor.c`
**位置**: 第135行

```c
// 修改前
#define FEEDFORWARD_GAIN  3.5f

// 修改后
#define FEEDFORWARD_GAIN  0.0f
```

**必要性**: 前馈控制适用于阶跃响应系统（目标突变），但扑动系统的目标是周期性平滑变化的。前馈会持续叠加输出，导致实际角度超过目标，造成超调。纯PID控制更适合周期性扑动。

**关键发现**: 这是V6.9的重要发现——扑动系统的目标角度按波形表规律平滑变化，不是突变信号。前馈在此场景下适得其反。

**功能目标**: 消除前馈导致的超调问题，让PID独立完成跟踪控制。

### 4.2 正确的调试步骤（按顺序执行）

#### 步骤1: 验证编码器数据通路

**目标**: 确认ADC→滤波→角度转换链路正确

**操作**:
1. 使用STM-Studio监控 `dbg_adc_raw_M3` 和 `dbg_adc_filtered_M3`
2. 手动转动电机，观察原始值和滤波值是否同步变化
3. 验证 `ALPHA_FLOAT=0.7` 下滤波延时约30ms
4. 验证 `LIMIT_THRESHOLD=200` 下无异常数据通过

**验证标准**: 手动转动时，滤波值应在30ms内跟上原始值，无突变和延迟

#### 步骤2: 验证波形表输出

**目标**: 确认BIOMIMETIC_WAVE[]输出正确

**操作**:
1. 使用STM-Studio监控 `dbg_wave_value`、`dbg_q15_result`、`dbg_current_idx`
2. 切换到Mode2，低油门(thr=6)
3. 观察 `dbg_current_idx` 是否在0-24之间循环
4. 观察 `dbg_wave_value` 是否从32767→0→-32767→0→32767对称变化

**验证标准**: 波形值严格对称，索引循环无跳变

#### 步骤3: 验证目标角度计算

**目标**: 确认q15_mul计算和目标角度范围正确

**操作**:
1. 监控 `dbg_M3_target` 和 `dbg_M3_angle`
2. 直飞状态(Yaw=0, midpoint=0)，观察目标角度范围
3. 验证：目标范围 = motor_midpoint ± amp ≈ 2000 ± 1024 = [976, 3024]

**验证标准**: 目标角度在合理范围内，无异常跳变

#### 步骤4: 验证PID跟踪性能

## 五、调试过程中必须理解和掌握的核心知识点

### 5.1 Q15定点数运算 ⭐⭐⭐

**概念**: Q15是16位定点数格式，1位符号+15位小数，范围[-32767, 32767]对应[-1.0, 1.0]。

**乘法规则**: `q15_mul(a, b) = (a × b) >> 15`

**在项目中的应用**:
```c
// 波形值cR是Q15格式（-32767~+32767）
// amp是编码器单位（如1024）
// q15_mul(amp, cR) = amp × cR / 32768
// 当cR=32767时，结果≈amp（满幅度）
// 当cR=0时，结果=0（中心位置）
// 当cR=-32767时，结果≈-amp（反方向满幅度）
```

**为什么用Q15**: STM32G0没有FPU，浮点运算慢。Q15定点乘法只需一次整数乘法+一次移位，比浮点快10倍以上。

### 5.2 非阻塞状态机设计 ⭐⭐⭐

**核心思想**: 不使用HAL_Delay()阻塞CPU，而是通过时间戳判断是否该执行下一步。

```c
// 错误做法（阻塞）
while (1) {
    execute_step();
    HAL_Delay(step_ms);  // CPU在这里空等！
}

// 正确做法（非阻塞）
while (1) {
    uint32_t now = HAL_GetTick();
    if ((int32_t)(now - sm_next_tick) >= 0) {
        execute_step();
        sm_next_tick += step_ms;  // 只记录下次时间，不阻塞
    }
    // CPU可以继续做其他事（ADC采样、PID计算等）
}
```

**关键**: `(int32_t)(now - sm_next_tick) >= 0` 这种写法能正确处理uint32_t溢出。

### 5.3 PID参数与频率的关系 ⭐⭐⭐

**核心原理**: 扑动频率越高，电机可用于响应的时间越短。

| 频率 | 每步可用时间 | PID策略 |
|------|------------|---------|
| 2.4Hz | ~17ms | 适度Kp(6.0)，确保跟踪 |
| 3.2Hz | ~12.5ms | 平衡Kp(5.5)，兼顾跟踪和稳定 |
| 4.0Hz | ~10ms | 保守Kp(5.0)，优先稳定 |

**为什么Ki=0**: 扑动是周期性运动，目标角度在正负之间来回变化。积分项会在一个方向累积，当目标反向时需要很长时间才能消除，导致"单向偏移"现象。

### 5.4 前馈控制 vs 纯PID在周期性系统中的差异 ⭐⭐

**前馈适用场景**: 目标突变（如阶跃响应），前馈可以"预判"方向提前输出

**前馈不适用场景**: 目标周期性平滑变化（如扑动），前馈持续叠加导致超调

```
阶跃系统：  目标 ──────┐          前馈：在阶跃瞬间给出大输出 ✅
                       │          PID：等误差出现才动作（慢）
                       
扑动系统：  目标 ∿∿∿∿∿∿∿∿  前馈：持续叠加 → 超过目标 → 超调 ❌
                           PID：跟踪误差 → 平滑跟随 ✅
```

### 5.5 滤波延时对控制系统的影响 ⭐⭐

**一阶低通滤波延时公式**: `延时 ≈ 采样周期 / α`

| α值 | 采样周期 | 延时 | 评价 |
|-----|---------|------|------|
| 0.1 | 2ms | 20ms | 滤波强，延时大 |
| 0.3 | 2ms | ~7ms | 平衡 |
| 0.5 | 2ms | 4ms | 响应快，滤波弱 |
| 0.7 | 2ms | ~3ms | 响应很快，滤波很弱 |

**关键**: 滤波延时会直接叠加到PID控制环路延时上。如果编码器反馈延时100ms，PID实际上是在100ms前的位置上做决策，必然导致超调。

### 5.6 模式切换时的状态管理 ⭐

**问题**: 从Mode2切换到Mode1/Mode0时，PID积分项可能累积了较大值。

**解决**: 
1. 模式切换时调用PID_clear()清零所有状态
2. 每次进入Mode1/Mode0时强制恢复PID参数
3. 重置扑动状态机(reset_flap_state)

### 5.7 波形表设计与电机能力匹配 ⭐⭐

**约束关系**:
```
波形表相邻点差值 → 每步角度变化 → 电机所需时间 → 必须小于MIN_STEP_MS
```

**V6.9设计验证**:
- 相邻差值: 5461 Q15单位
- amp=1024时角度变化: 1024 × 5461 / 32768 ≈ 171 编码器值 ≈ 15°
- 电机所需时间: 15° × 0.83ms/° ≈ 12.5ms
- MIN_STEP_MS = 8ms ⚠️ 临界！

**注意**: 当前MIN_STEP_MS=8ms对于amp=1024来说偏小，实际运行中amp会被幅度补偿减小（高频时0.91x≈932），对应约14°，需11.6ms。如果发现跟踪不良，应将MIN_STEP_MS增大到10-12ms。

### 5.8 对称控制与电机镜像安装 ⭐⭐

**物理布局**（从蝴蝶后方看）:
```
         左前(M3)     右前(M1)
```

**对称原则**:
- 左侧电机（M3）：使用 +(PID+FF) 正输出
- 右侧电机（M1）：使用 -(PID+FF) 负输出（整体取反）

**原因**: 左右电机镜像对称安装，相同PWM导致相反物理运动。要让两翅同时向下拍，右侧必须取反。
