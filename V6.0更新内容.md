## 一、系统概述

本项目是一个基于STM32G0的**仿生蝴蝶四翼飞行器控制系统**，采用磁编码器反馈、PID闭环控制，实现了三种飞行模式：并翅模式、平翅模式和扑动模式。

### 1.1 硬件配置

```
主控芯片：STM32G0系列
电源：7.4V 锂电池（20C放电倍率）
电机：615空心杯电机 + 136:1减速比 + 3级塑料齿轮箱
    ├── 空载转速：360 RPM (6Hz) @ 7.4V
    ├── 堵转扭矩：≥400 gf·cm
    └── 最大承受扭矩：400 gf·cm（超过会损坏！）
传感器：AS5600磁编码器（12位精度，0-4095）
接收机：ELRS（CRSF协议）
```

### 1.2 软件架构

```
main.c (主循环)
    ├── ELRS模块（遥控器数据接收）          → [elrs.c](user/module/elrs/elrs.c)
    ├── AS5600_PWM模块（磁编码器角度采集）   → [AS5600_PWM.c](user/module/AS5600/AS5600_PWM.c)
    ├── flight_control模块（飞行控制算法）   → [flight_control.c](user/module/flight_control/flight_control.c) ⭐核心
    ├── motor模块（电机PID控制）            → [motor.c](user/module/motor/motor.c)
    └── pid模块（PID算法库）                → [pid.c](user/module/pid/pid.c)
```

## 三、🔧 调试指南（重要！）

### 3.1 首次调试流程

#### 步骤1：检查硬件连接 ✅
```
□ 4个电机都能正常转动（用motor_test()函数测试）
□ 4个编码器都有数值变化（观察debug_current[]数组）
□ ELRS接收机连接正常（观察elrs_data.Switch是否随开关变化）
□ 电源电压7.4V正常
```

#### 步骤2：校准电机中点值 🎯
**这是最关键的一步！**

```c
// 文件：AS5600_PWM.h 第30-33行
// 方法：
// 1. 将所有电机手动调整到水平位置（翅膀完全展开）
// 2. 读取此时的AD_Value[0-3]原始值
// 3. 将这4个值填入下面的宏定义
#define MOTOR1_MIDPOINT XXXX  // 替换为实测值
#define MOTOR2_MIDPOINT XXXX
#define MOTOR3_MIDPOINT XXXX
#define MOTOR4_MIDPOINT XXXX
```

#### 步骤3：测试Mode0（并翅模式）🔧
```
操作：拨杆到Mode0
预期：所有电机缓慢归位到1024（竖直位置）

如果出现问题：
  □ 电机不动 → 检查Kp是否太小（当前=8）
  □ 电机抖动 → 检查中点值是否正确
  □ 归位太慢 → 增大MAX_VELOCITY（当前=60）
  □ 冲击太大 → 减小ACCEL_LIMIT（当前=8）
```

**Mode0可调参数**（[main.c第260-290行](Core/Src/main.c#L260-L290)）：

| 参数 | 位置 | 作用 | 调整方向 |
|------|------|------|----------|
| `SMOOTH_INTERVAL` | 5ms | 更新周期 | 减小=更快响应 |
| `MAX_STEP` | 60 | 最大速度 | 增大=更快归位 |
| `ACCEL_LIMIT` | 8 | 加速度限制 | 减小=更平滑 |
| `HARD_DEADZONE` | 10 | 硬死区 | 增大=提前停止 |
| `SOFT_DEADZONE` | 40 | 软死区 | 增大=提前减速 |

#### 步骤4：测试Mode1（平翅模式）✈️
```
操作：拨杆到Mode1
预期：翅膀保持水平，能响应Yaw转向

如果出现问题：
  □ 不水平 → 检查midpoint微调（遥控器通道1）
  □ 转向不灵敏 → 增大Yaw系数（当前=2×）
  □ 抖动 → 降低Kp（当前=10）
```

#### 步骤5：测试Mode2（扑动模式）🦋
```
操作：拨杆到Mode2，油门从小到大慢慢推
预期：开始缓慢扑动，逐渐加快

首次测试建议：
  ✓ 油门从5开始（最慢）
  ✓ 观察1-2分钟确认无异常
  ✓ 逐步增加到8-10
  ✓ 监测电机温度（手摸不烫手）

如果出现问题：
  □ 有"咔咔"冲击声 → 增大BUFFER_ZONE_SIZE（当前=100→150）
  □ 扑动幅度太小 → 增大MAX_AMP（当前=1100→1200）
  □ 扑动太慢 → 减小MIN_STEP_MS（当前=16→12）
  □ 升力不足 → 增大FLAP_CENTER_OFFSET（当前=200→250）
  □ 电机发热 → 降低MAX_AMP或MAX_BIOMIMETIC_FREQ
```

### 3.3 波形表自定义 🎨

如果你想自定义扑动波形，修改[flight_control.c第16-47行](user/module/flight_control/flight_control.c#L16-L47)的`BIOMIMETIC_WAVE`数组：

```c
// 当前：25点非对称波形（推荐）
static const int16_t BIOMIMETIC_WAVE[25] = {
    // 下拍阶段（快速）
    32767, 32000, 29491, 24500, 18204, 11300, 
     6393,     0, -6393,-12540,-18204,-23170,-27146,
    // 极限缓冲
   -28900,
    // 上挥阶段（缓慢）
   -26384,-22986,-18794,-13856, -8200, -3000,
     2500,  8000, 13500, 20000, 28000
};
```

**自定义要点**：
- 数值范围：-32768 到 +32767（Q15格式）
- +32767 = 上极限位置
- -32767 ≈ 下极限位置（建议不小于-30000以保留缓冲）
- 数组长度可以改变（17/25/33点），但要同步修改状态机逻辑

## 五、常见问题排查

### 问题1：电机不转或乱转 🔴

**检查清单**：
1. ✅ 编码器中点值是否正确？（最常见原因）
2. ✅ PWM接线是否正确？（M1-M4对应关系）
3. ✅ 电机正反转是否需要调整？

**排查方法**：
```c
// 在main.c开头调用这个测试函数
motor_test();  // 观察电机转向是否正确
```

---

### 问题2：扑动时有"咔咔"声 🟡

**原因**：到达极限位置时冲击过大

**解决方案**（按顺序尝试）：
1. 增大`BUFFER_ZONE_SIZE`到150
2. 降低`MAX_AMP`到1000
3. 如果仍有声音，降低`MAX_BIOMIMETIC_FREQ`到5

---

### 问题3：飞机飞不稳/偏航 🟠

**可能原因**：
1. 4个电机不同步
2. 中点校准不准
3. 机械安装不对称

**解决方案**：
1. 检查`SYNC_GAIN`是否足够（当前=5）
2. 重新校准中点值
3. 观察debug_err[]数组，看哪个电机误差大

---

### 问题4：升力不足 🔵

**调整方案**：
1. 增大`FLAP_CENTER_OFFSET`到250
2. 增大`MAX_AMP`到1200
3. 加快下拍：`DOWNSTROKE_RATIO`=35

---

### 问题5：电机过热发烫 🔴

**紧急处理**：
1. 立即停止运行！
2. 降低以下参数：
   - `MAX_AMP`: 1100 → 900
   - `MAX_BIOMIMETIC_FREQ`: 6 → 4
   - `Kp`: 15 → 10

**预防措施**：
- 连续运行不超过5分钟
- 手摸电机外壳监测温度
