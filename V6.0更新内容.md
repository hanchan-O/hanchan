## 一、系统概述

本项目是一个基于STM32G0的**仿生蝴蝶四翼飞行器控制系统**，采用磁编码器反馈、PID闭环控制，实现了三种飞行模式：并翅模式、平翅模式和扑动模式。

### 1.1 硬件配置

```
主控芯片：STM32G0系列
电源：7.4V 锂电池（20C放电倍率）
电机：615空心杯电机 + 136:1减速比 + 3级塑料齿轮箱
    ├── 空载转速：360 RPM (6Hz) @ 7.4V
    ├── 堵转扭矩：≥400 gf·cm
    └── 最大承受扭矩：400 gf·cm（超过会损坏！）
传感器：AS5600磁编码器（12位精度，0-4095）
接收机：ELRS（CRSF协议）
```

### 1.2 软件架构

```
main.c (主循环)
    ├── ELRS模块（遥控器数据接收）          → [elrs.c](user/module/elrs/elrs.c)
    ├── AS5600_PWM模块（磁编码器角度采集）   → [AS5600_PWM.c](user/module/AS5600/AS5600_PWM.c)
    ├── flight_control模块（飞行控制算法）   → [flight_control.c](user/module/flight_control/flight_control.c) ⭐核心
    ├── motor模块（电机PID控制）            → [motor.c](user/module/motor/motor.c)
    └── pid模块（PID算法库）                → [pid.c](user/module/pid/pid.c)
```

### 2.4 ⚠️ 特殊处理说明

#### **V6.2：编码器方向已统一（不再需要反转！）**
```
V6.2之前的处理：
  • 电机2(左前)、电机3(右后)的编码器安装方向与其他相反
  • 需要用 4096-raw_value 反转

V6.2的改进：
  ✅ 删除了所有4096-x反转代码
  ✅ 要求用户统一安装所有4个编码器的磁铁方向
  ✅ 统一标准：顺时针转 → 编码器值增大 → 翅膀向下

  如果某个编码器方向反了：
  → 把那个编码器的磁铁旋转180°重新安装即可
  → 不需要在代码中做任何特殊处理

  详细接线要求见 → [三、📌 硬件接线指南](#三硬件接线指南v62新增)
```

#### **为什么右侧电机需要PID输出取反？**
```
原因：左右电机是镜像对称安装的蝴蝶翅膀

从后方观察蝴蝶：
  • 左侧电机（左前+左后）：逆时针转动 → 翅膀下拍
  • 右侧电机（右前+右后）：顺时针转动 → 翅膀下拍
  
代码实现：
  • 左侧：直接使用 +(PID_output + feedforward)
  • 右侧：使用 -(PID_output + feedforward) 整体取负
  
效果：
  • 相同的目标角度变化 → 左右电机产生相反的物理转动
  • 实现协同扑动（同时向下或同时向上）
```

#### **为什么Set_Pwm参数顺序是 (m1, m4, m3, m2)？**
```
原因：为了匹配实际硬件PCB接线！

【V6.2完整映射 - 按此接线！】

PWM变量              Set_Pwm参数    硬件驱动器    STM32引脚       物理电机
────────────────────────────────────────────────────────────────────
motor_1_set_pwm  →   第1个(m1)  →   M1驱动器  → PA15+PB3   →  右前
motor_4_set_pwm  →   第2个(m2)  →   M2驱动器  → PA2+PA3    →  右后
motor_3_set_pwm  →   第3个(m3)  →   M3驱动器  → PB4+PB5    →  左前
motor_2_set_pwm  →   第4个(m4)  →   M4驱动器  → PB0+PB1    →  左后

⚠️ 这个顺序是固定的，与PCB布线一致！
详细接线图见 → [三、📌 硬件接线指南](#三硬件接线指南v62新增)
```

### 2.5 🛠️ 调试检查清单

当出现以下问题时，按此表排查：

| 故障现象 | 检查层级 | 排查方法 |
|---------|---------|----------|
| **某个电机不动** | ①②③ | 检查`debug_current[i]`是否有数值变化 |
| **电机转向相反** | ②⑨ | 检查编码器反转或PID取反是否正确 |
| **左右翅不同步** | ⑨ | 检查右侧电机是否正确取反 |
| **电机抖动严重** | ④⑤ | 检查中点值是否校准准确 |
| **跟踪误差大** | ⑥⑦ | 检查PID参数Kp/Ki/Kd是否合理 |
| **冲击噪音大** | ⑥ | 检查BUFFER_ZONE_SIZE是否足够大 |
| **只有一个翅动** | ⑩⑪ | 检查Set_Pwm参数是否正确传递 |

#### **推荐调试步骤**
```
步骤1：静态测试（Mode0并翅模式）
  ├─ 观察 debug_current[0-3] 是否都有数值
  ├─ 手动转动每个电机，确认数值变化方向一致
  └─ 如果某个电机反向 → 检查第②层的编码器反转

步骤2：单电机测试
  ├─ 注释掉其他3个电机的PID计算
  ├─ 只让一个电机运行，观察运动方向
  └─ 如果方向错误 → 检查第⑨层的取反逻辑

步骤3：对称性测试
  ├─ 给左右电机相同的目标角度
  ├─ 观察 debug_err[0] 和 debug_err[2] 应该近似相反数
  └─ debug_err[0] ≈ -debug_err[2] （右前 vs 左前）

步骤4：全系统联调
  ├─ Mode2扑动模式，油门从最小开始
  ├─ 观察4个电机的运动是否协调同步
  └─ 监听有无异常声音或振动
```

步骤2：编码器方向验证  
  手动逐个压下翅膀，观察AD_Value变化
  4个都应该：下压 → 数值增大

步骤4：对称性验证
  Mode=1平翅模式，给Yaw指令
  左侧和右侧翅膀应该向相反方向倾斜（对称运动）

#### 🔥 V6.1核心改进（2026-04更新）

| 改进项 | V6.0旧方案 | **V6.1新方案** |

| **前馈控制** | 无 | **PD + 前馈补偿（速度预测）** |
| **输出保护** | 仅PID限幅 | **PID+前馈叠加后再次限幅** |

##### 📌 PID参数定义位置
```c

    {18.0f, 0.08f, 3.0f},     // 电机1(左后)
    {18.0f, 0.08f, 3.0f},     // 电机2(左前)
    {18.0f, 0.08f, 3.0f}      // 电机3(右后)
};
```

##### 📌 差异化策略（main.c动态切换）

| 模式 | 触发条件 | Kp | Ki | Kd | 适用场景 |
|------|---------|-----|-----|-----|---------|
| **Mode 0** | 并翅定位 | 10.0 | 0.05 | 2.0 | 静态稳定性优先 |
| **Mode 1** | 简单平翅 | 10.0 | 0.05 | 2.0 | 平衡响应与稳定 |
| **Mode 2** | 高速扑动 | **18.0** | **0.08** | **3.0** | 快速跟踪优先 |

> 💡 **设计思路**：Mode 0/1使用保守参数防止抖动，Mode 2使用激进参数提高扑动跟踪精度。只在模式切换时设置一次，节省CPU资源。

##### 📌 其他关键参数

| 参数名 | 当前值 | 位置 | 作用 | 调整建议 |
|--------|--------|------|------|----------|
| 输出限幅 `motore_max_out` | 12000 | [motor.c:44](user/module/motor/motor.c#L44) | PWM最大输出 | 不要超过15000 |
| 积分限幅 `motor_max_iout` | 4095 | [motor.c:45](user/module/motor/motor.c#L45) | 积分项限制 | 保持不变 |
| 前馈增益 `FEEDFORWARD_GAIN` | 3.5 | [motor.c:200+](user/module/motor/motor.c) | 前馈强度 | 减小到2.5更温和 |

#### 左右对称控制详解

```
物理布局（从后方看蝴蝶）：
        ┌─────────────────┐
        │   左前(2) 右前(0) │
        │                   │
        │   左后(1) 右后(3) │
        └─────────────────┘

控制逻辑：
├── 左侧电机（左前M3 + 左后M4）
│   └── 输出 = +(PID_output + feedforward)  ← 正号，正常方向
│
└── 右侧电机（右前M1 + 右后M2）
    └── 输出 = -(PID_output + feedforward)  ← 负号，镜像对称！

效果：相同目标角度变化时，
     左侧逆时针转，右侧顺时针转 → 协同扑动！
```

## 七、常见问题排查

### 问题1：电机不转或乱转 🔴

**检查清单**：
1. ✅ 编码器中点值是否正确？（最常见原因）
2. ✅ PWM接线是否正确？（M1-M4对应关系）
3. ✅ 电机正反转是否需要调整？

**排查方法**：
```c
// 在main.c开头调用这个测试函数
motor_test();  // 观察电机转向是否正确
```

---

### 问题2：扑动时有"咔咔"声 🟡

**原因**：到达极限位置时冲击过大

**解决方案**（按顺序尝试）：
1. 增大`BUFFER_ZONE_SIZE`到150
2. 降低`MAX_AMP`到1000
3. 如果仍有声音，降低`MAX_BIOMIMETIC_FREQ`到5

---

### 问题3：飞机飞不稳/偏航 🟠

**可能原因**：
1. 4个电机不同步
2. 中点校准不准
3. 机械安装不对称

**解决方案**：
1. 检查`SYNC_GAIN`是否足够（当前=5）
2. 重新校准中点值
3. 观察debug_err[]数组，看哪个电机误差大

---

### 问题4：升力不足 🔵

**调整方案**：
1. 增大`FLAP_CENTER_OFFSET`到250
2. 增大`MAX_AMP`到1200
3. 加快下拍：`DOWNSTROKE_RATIO`=35

---

### 问题5：电机过热发烫 🔴

**紧急处理**：
1. 立即停止运行！
2. 降低以下参数：
   - `MAX_AMP`: 1100 → 900
   - `MAX_BIOMIMETIC_FREQ`: 6 → 4
   - `Kp`: 15 → 10

**预防措施**：
- 连续运行不超过5分钟
- 手摸电机外壳监测温度
